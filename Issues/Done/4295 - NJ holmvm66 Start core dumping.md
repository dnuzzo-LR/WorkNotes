# Issue 4295 — NJ holvm66 - "Start" core dumping

[View on GitHub](https://github.com/lightriversoftware/netflex/issues/4295)

## Summary

On the NJ BEP `holmvm66` (s/w 53.0.29), the `start` process core dumps (SIGSEGV) when bringing up the application. The crash is in `get_PM_tunables()` → `strcmp` against "PM DESCHEDULE DAYS" with a corrupt pointer, so it looked PM-tunables related. (Separately, on first boot the CMM password was prompted because authorized keys were missing for cmm; that was corrected before the core-dump investigation.)

## Observed Behavior

`gdb` backtrace of `/usr/cnc/bin/start` core: SIGSEGV in `__strcmp_avx2` → `strcmp` (a = invalid address) from `get_PM_tunables()` (pm_sup.c:4156) called by `main` (start.c:618). Core generated by `/usr/cnc/bin/start holmvm66 15000 MultiBox Active`.

## Resolution / Notes

- **dnuzzo-LR (2026-05-27):** Root cause was a `libutillib.so` that **did not match the installed patch**. Mismatched library → the segfault in the PM tunables path.
