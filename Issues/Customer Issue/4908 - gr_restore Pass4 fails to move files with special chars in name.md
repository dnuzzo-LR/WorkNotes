# Issue 4908 — [SUPPORT] gr_restore Pass[4] fails to move files in wvlsvc_match_del_logs/ with ( ) , < > or space in name (5.4.026)

[View on GitHub](https://github.com/lightriversoftware/netflex/issues/4908)

## Summary

On every 5.4.026 GR (geographic redundancy) transfer, the `gr_restore` file-move pass (Pass[4]) fails to move files in `/usr/cnc/data/wvlsvc_match_del_logs/` whose filenames contain parentheses, commas, `<-`/`->` arrows, or spaces. Each failure logs `gr_restore.cpp:449 ERROR: Pass[4]: Failed to move ...`. The STANDBY ends up with an incomplete wvlsvc-match-delete log directory after every transfer. Core data restore (provisioning, core, tie.dat, dno*.dat) completes cleanly — only this feature-file move step fails.

These filenames are generated by netFLEX itself from operator-defined wavelength-service names and are syntactically valid POSIX paths. Manually renaming a problem file (e.g. replacing `<->` with `__`, or stripping spaces/parens) lets it move cleanly on the next transfer and it does not re-fail — pointing at how Pass[4] invokes the move per-filename rather than a filesystem limitation.

## Reproduction Steps

1. On a 5.4.026 GR pair, ensure files exist in `/usr/cnc/data/wvlsvc_match_del_logs/` whose names contain `(`, `)`, `,`, `<-`/`->`, or space (netFLEX produces these from operator-defined wavelength-service names).
2. Trigger a GR transfer.
3. Tail the `gr_restore` output on the STANDBY — each affected filename logs a `Pass[4]: Failed to move` error at `gr_restore.cpp:449`.

## Observed Behavior

- Every failing line: `gr_restore.cpp|449|ERROR: Pass[4]: Failed to move <features>/data/wvlsvc_match_del_logs/<FILENAME> to /usr/cnc/data/wvlsvc_match_del_logs/<FILENAME>`.
- Charter Golden OS (holmvm75 ↔ holmvm76): 1,988 errors across 5 transfers since 2026-06-10 (individual runs of 814 and 943 errors). Lab (holmvm17 ↔ linc12vm9): 15 errors in one transfer, same character classes.
- Filename character-class breakdown (holmvm76, 1,988 errors): `(`/`)` 1,402; `<->` 457; space 115; other/combinations 14.
- Impact: STANDBY missing 100s–1000s of files per transfer; incomplete forensic history on failover; large persistent "GR Restore Completed With N Errors" alarms drowning real failure signals. Does NOT affect NE-traffic correctness or core RDB data.
